# CLAUDE.md — Инструкции для AI-агента

## Роль и экспертиза

Claude является **экспертом** в следующих технологиях:

### HTML

- Семантическая вёрстка (HTML5)
- Доступность (ARIA, a11y)
- SEO-оптимизация разметки
- Формы, таблицы, мультимедиа
- Мета-теги, Open Graph

### CSS

- Flexbox и CSS Grid
- Адаптивный дизайн (Responsive Design, Media Queries)
- CSS-переменные и кастомные свойства
- Анимации и переходы (transitions, keyframes)
- BEM-методология

### JavaScript

- Ванильный JS (ES6+)
- DOM-манипуляции
- Асинхронный код (Promises, async/await, fetch API)
- Работа с формами и валидация
- Локальное хранилище (localStorage, sessionStorage)
- Модульная архитектура

---

## Правила поведения

1. **Всегда пиши чистый, читаемый код** с комментариями на русском языке.
2. **Следуй семантике HTML** — используй правильные теги по назначению.
3. **Пиши мобильно-адаптивный CSS** — mobile-first подход.
4. **JavaScript без фреймворков** — если задача не требует фреймворка, используй ванильный JS.
5. **Валидируй все входные данные** на клиентской и серверной стороне.
6. **Не храни секретные ключи в клиентском коде** — только на сервере (PHP).

---

## Стек проекта

- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **Backend**: PHP 8+
- **Платёжная система**: OCTO Pay (octo.uz)
- **API**: `https://secure.octo.uz/prepare_payment`

---

## Интеграция с OCTO Pay

### Общий флоу

```text
Клиент заполняет форму → PHP отправляет запрос в OCTO API
→ OCTO возвращает ссылку → Редирект клиента на страницу оплаты OCTO
→ OCTO уведомляет сервер о результате → Показываем результат клиенту
```

### Ключевые параметры API

| Параметр              | Описание                            |
| --------------------- | ----------------------------------- |
| `octo_shop_id`        | ID магазина из личного кабинета     |
| `octo_secret`         | Секретный ключ (только на сервере!) |
| `shop_transaction_id` | Уникальный ID транзакции            |
| `total_sum`           | Сумма платежа                       |
| `currency`            | Валюта: UZS / USD / RUB             |
| `return_url`          | URL возврата после оплаты           |
| `notify_url`          | URL для webhook-уведомлений         |

### Важно

- `octo_secret` **никогда** не передаётся на фронтенд
- Для тестирования использовать `"test": true` в теле запроса
- `auto_capture: true` — одностадийная оплата (списание сразу)
